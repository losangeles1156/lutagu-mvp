# ルタグ（LUTAGU）介面規格
# UI/UX 設計規範 - 前端開發指引

> **產品名稱**：ルタグ（LUTAGU）
> **內部代號**：LUTAGU
> **版本**：v3.2

---

## 🎯 本文件的使用方式

```
給前端開發者 / UI 設計師的指引：

1. 採用「底部四分頁」單頁應用架構
2. 四個 Tab 對應 L1-L4：DNA / LIVE / FACILITY / LUTAGU
3. L1 顯示景點主要營業服務，有導航按鈕
4. L3 MVP 分兩區：車站設施（無導航）+ 外部服務連結
5. L4 是完整的行程規劃助手，包含使用者狀態選擇
6. 外部連結需追蹤點擊，用於產品驗證
```

## 4. FACILITY 分頁設計（L3 服務設施）

### 4.1 功能定位

```
L3 FACILITY 回答的問題：「我需要 X，哪裡有？」

MVP 架構：
• Section A：車站設施（電梯、廁所、育嬰室等）
• Section B：外部服務連結（置物櫃、ecbo cloak、共享單車等）

特點：
• 車站設施只顯示相對位置，無導航按鈕
• 外部連結導向第三方服務，追蹤點擊
• 無障礙設施放在最前面

│   • 當前車站節點                                                 │
│   • 連結類型（locker / ecbo / bike / taxi）                     │
│   • 用戶 Session ID                                             │
│                                                                 │
│   用途：                                                         │
│   • 分析用戶對各服務的需求程度                                   │
│   • 決定未來優先整合哪些 API                                     │
│   • 例：如果 ecbo cloak 點擊率最高 → 優先整合 ecbo API           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.4 未來擴展（非 MVP）

```
未來擴展：Section C - 周邊店家設施

當有足夠數據或合作夥伴時，可加入：
• 哪家咖啡廳有充電插座
• 哪家便利商店有 ecbo cloak
• 哪家百貨有免費 WiFi

這些店家會同時出現在 L1（主營業務）和 L3（附加服務）
```
