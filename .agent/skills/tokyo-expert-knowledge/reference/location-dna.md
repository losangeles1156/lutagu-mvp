# Location DNA & Characteristics (地點特性)

本文件定義地區的性格 (Persona)、氛圍匹配邏輯與時空特性。
目標是回答：「這裡是什麼感覺？」與「有沒有類似的地方？」

## 1. 地點 DNA 標籤 (Location DNA Tags)

基於 L1 Tagging System，但更側重於「區域性格」而非單點功能。

| 標籤 | 定義 | 代表區域 |
| :--- | :--- | :--- |
| **#下町 (Shitamachi)** | 傳統、懷舊、人情味、窄巷 | 淺草、谷中銀座、柴又、人形町 |
| **#商業區 (Business)** | 高樓、西裝族、整潔但冷漠 | 丸之內、大手町、品川、汐留 |
| **#次文化 (Subculture)** | 動漫、古著、獨立音樂、混亂 | 秋葉原、下北澤、高圓寺、中野 |
| **#高大上 (Upscale)** | 精品、寬闊街道、高級感 | 銀座、表參道、六本木 |
| **#文教區 (Academic)** | 學校多、舊書店、安靜咖啡廳 | 神保町、本鄉三丁目、早稻田 |

## 2. 氛圍相似度匹配 (Vibe Similarity)

**應用場景**: 「我很喜歡淺草，有沒有類似的地方？」
**匹配邏輯**:
*   若 User 喜歡 `淺草` (#下町 + #觀光)，推薦 -> `川越` (小江戶)、`柴又`。
*   若 User 喜歡 `代官山` (#精緻 + #散步)，推薦 -> `自由之丘`、`中目黑`。
*   若 User 喜歡 `秋葉原` (#動漫 + #電器)，推薦 -> `中野 Broadway` (更深度)。

**AI 提示語**:
> 「如果你喜歡淺草的熱鬧傳統感，那你一定會愛上**柴又**！它更有電影場景的老街味道，但觀光客少很多。」

## 3. 時段特性 (Time-based Characteristics)

**應用場景**: 「什麼時候去比較不擠？」

*   **通勤地獄 (Rush Hour)**: 平日 7:30-9:30, 17:30-19:30。
    *   *避開*: 大手町、新宿、品川等轉乘大站。
*   **午後雷區**: 平日 12:00-13:00 (商業區午餐時間)。
    *   *現象*: 餐廳大排長龍。
*   **週末大變身**:
    *   *丸之內/大手町*: 週末反而空曠 (因為上班族不在)。
    *   *原宿/竹下通*: 週末是「災難級」擁擠，建議平日上午去。

## 4. 季節與人潮戰略 (Seasonal & Crowd Strategy)

**應用場景**: 「這個週末會很擠嗎？下雨怎麼辦？」

### 4.1 避開雷區 (Crowd Avoidance)
*   **通勤地獄 (Rush Hour)**: 平日 7:30-9:30。**東西線 (Tozai Line)** 是絕對禁區 (混雜率 199%)。
*   **週末大變身**:
    *   *丸之內/大手町*: 週末反而空曠 (因為上班族不在)，適合悠閒散步。
    *   *原宿/竹下通*: 週末是「災難級」擁擠，建議平日上午去。

### 4.2 氣候應變 (Weather Tactics)
*   **突發暴雨 (Guerrilla Rain / ゲリラ豪雨)**: 夏季午後常見。
    *   *避難所*: 立即躲入地下街 (八重洲地下街、新宿 Subnade)。
    *   *備案*: 安排「室內備案」 (百貨、博物館) 以備不時之需。

### 4.3 重大活動 (Major Events)
*   **花火大會 (7-8月)**: 隅田川花火 (淺草/押上)、神宮外苑花火。
    *   *影響*: 車站下午就開始管制，建議提早 3 小時到達或直接避開該區域。
*   **年末年始 (12/29-1/3)**:
    *   *現象*: 商辦區像死城，神社 (明治神宮、淺草寺) 人潮爆滿 (初詣)。
