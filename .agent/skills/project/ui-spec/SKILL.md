# LUTAGU UI/UX 設計規範

本 Skill 提供 LUTAGU (ルタグ) PWA 應用程式的核心介面架構與交互原則。

## 🎯 核心架構：四分頁單頁應用 (SPA)

應用程式採用底部導航，對應數據層級 L1-L4：

1.  **DNA (L1)**: 地點基因。回答「我附近有什麼？」。顯示景點主要服務與導航。
2.  **LIVE (L2)**: 即時狀態。回答「現在狀況如何？」。包含運行情報、天氣、人潮。
3.  **FACILITY (L3)**: 服務設施。回答「我需要 X，哪裡有？」。分車站設施與外部連結。
4.  **LUTAGU (L4)**: 行動建議。回答「我該怎麼做？」。個人化行程規劃助手。

## 🚀 核心設計原則

-   **無障礙優先**: L3 設施與 L4 狀態選擇均優先考慮無障礙需求 (🦽, 👶)。
-   **行動導向 (Nudge)**: 介面不僅提供資訊，更要引導使用者採取行動（如：按下導航、預約單車）。
-   **單一推薦**: 輸出應收斂至單一「主要建議卡片」。
-   **商業導向**: 在交通擁塞時，主動顯示替代方案 (Uber, LUUP)。

## 🎨 頂部共用區域

所有分頁均包含：
-   **節點名稱與標籤**: 例如「上野 · CULTURE」。
-   **天氣資訊**: 即時溫度與圖示。
-   **地圖預覽**: 可摺疊，點擊可進入大圖。
-   **JMA 警報**: 紅色橫幅顯示緊急天氣資訊。
