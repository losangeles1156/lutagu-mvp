### 1.2 LIVE 分頁 (L2)
包含區塊：列車運行情報、天氣狀況、緊急警報、人潮狀況 (五級回報系統)。

### 1.3 FACILITY 分頁 (L3)
- **Section A**: 車站設施（電梯、廁所、育嬰室）。無導航。
- **Section B**: 外部服務連結（置物櫃、共享單車、計程車）。

### 1.4 LUTAGU 分頁 (L4)
行程規劃助手，包含：
- 使用者狀態選擇 (🦽, 👶, 🧳, ⏰, 💰, 🌧️)
- 即時情境摘要
- 主要建議卡片（專家知識提醒 + 客製化建議）

## 2. 數據細分與邏輯

### 2.1 住宿類別 (L1)
- `business_hotel`: 平價商務 (APA, 東横INN)
- `luxury_hotel`: 星級飯店 (帝國飯店)
- `hostel`: 青年旅館
- `capsule_hotel`: 膠囊旅館
- `onsen_ryokan`: 溫泉旅館
- `serviced_apartment`: 長住公寓

### 2.2 使用者狀態對建議的影響 (L4)
- **輪椅/嬰兒車**: 過濾無電梯路線。
- **大行李**: 避開尖峰時段，過濾只有樓梯的出口。
- **趕時間**: 顯示最快路線。
- **避雨**: 優先地下通道。

## 4. 設計原則 (詳細版)
1. **MVP 聚焦**: L1 聚焦景點主營業務，L3 聚焦車站與外部連結。
2. **導航按鈕規則**: 僅 L1 具體景點卡片提供導航。
3. **數據驅動**: 追蹤 Section B 外部連結點擊以決定未來 API 整合優先級。
