# LUTAGU L4 「完整行程規劃助手」演示指南 (Demo Guide)

本文件說明如何使用 LUTAGU MVP 的 L4 演示腳本來展示系統的核心功能與專家知識整合。

## 1. 演示腳本概覽

我們設計了四個核心演示情境，涵蓋了旅遊過程中的常見痛點：

### 情境 01：解決過度觀光 (Overtourism)
- **問題**：淺草寺人潮過多。
- **展示重點**：
    - 基於使用者感受的在地避人建議。
    - 專家精選方案（今宮神社、根津神社、柴又帝釋天）。
    - 互動式引導：提供轉乘標註或人力車查詢。

### 情境 02：交通中斷應變 (Disruption)
- **問題**：中央線延誤，如何前往東京都廳。
- **展示重點**：
    - 實時異常偵測（模擬 L2 狀態）。
    - **專家最優決策**：推薦西新宿站而非新宿站（出口更簡單、步行更近）。
    - 避開複雜大站的策略性引導。

### 情境 03：實踐空手觀光 (Hands-free)
- **問題**：成田機場抵達後行李寄放問題。
- **展示重點**：
    - 外部實時連結整合（東京 Metro 置物櫃狀態）。
    - 多樣化替代方案：Ecbo Cloak (店鋪寄存) 與 LUGGAGENT (機場直送)。
    - 第三方服務連結一鍵跳轉。

### 情境 04：無障礙路線規劃 (Accessibility)
- **問題**：推嬰兒車前往上野動物園的最佳出入口。
- **展示重點**：
    - ODPT 設施數據整合。
    - **精確路徑指引**：直接鎖定「上野站公園口」（平地層、無電梯需求）。
    - 嬰兒車/輪椅友善動線驗證。

## 2. 操作說明

1. **開啟 L4 儀表板**。
2. **展開「預設問題模板」**。
3. **選擇「系統特色 (Features)」類別**。
4. **點擊任一「演示：...」按鈕**。
5. **互動流程**：
    - 系統會顯示第一輪對話。
    - 點擊下方的 **「點擊模擬使用者後續提問」** 按鈕，展示異常處理或深度解釋。
    - 觀察系統如何標註使用的工具（如 `🛠️ 專家知識庫`）與執行的動作。

## 4. 技術架構說明

- **數據層**：`demoScenarios.ts` 定義靜態腳本內容。
- **邏輯層**：`AssistantEngine.ts` 負責識別演示觸發詞。
