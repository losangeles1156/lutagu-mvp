# Test Plan - Phase 3: L4 Station Assistant (Shinjuku)

## 測試目標
驗證進入特定車站（以新宿站為例）的 L4 深度頁面後，AI 助手的導航路徑、初始載入、對話功能與回應正確性。

## 測試路徑
1. 導覽至首頁。
2. 搜尋「新宿」並點擊進入車站詳情。
3. 切換至「智能嚮導」(LUTAGU Tab/L4)。
4. 點擊「開始對話」進入 L4 Chat。

## 測試場景 (Scenarios)

### Scenario 1: L4 進入路徑與初始載入 (P0)
- **動作**: 進入新宿站 L4 頁面。
- **預期結果**:
  - 成功切換至 L4 Dashboard。
  - 顯示「AI 智慧感知已啟動」狀態。
  - 顯示該車站特有的專家知識（陷阱提醒、攻略）。

### Scenario 2: 啟動 L4 AI 助手 (P0)
- **動作**: 點擊「開始對話」按鈕。
- **預期結果**:
  - 進入 `L4_Chat` 介面。
  - 顯示初始歡迎訊息（包含車站名稱「新宿」）。
  - 顯示建議問題列表。

### Scenario 3: L4 深度對話驗證 (P1)
- **動作**: 輸入特定問題，例如「新宿車站有寄物櫃嗎？」。
- **預期結果**:
  - 顯示轉圈圈 (Loader) 與思考步驟 (Thinking Step)。
  - 收到流式回覆。
  - 回覆內容包含車站設施相關資訊或引導。

### Scenario 4: 切換模式與視窗控制 (P2)
- **動作**: 測試縮小、放大或重置對話。
- **預期結果**:
  - 重置後對話回到初始狀態。
  - 視窗模式切換正常。

## 技術細節
- **Station ID**: `odpt:Station:JR-East.Shinjuku`
- **主要組件**: `NodeTabs`, `L4_Dashboard`, `L4_Chat`
- **關鍵文字標線 (Locators)**:
  - 搜尋框: `[placeholder="搜尋車站、路線..."]`
  - LUTAGU Tab: `role="tab" >> text="智能嚮導"`
  - 開始對話按鈕: `button >> text="開始對話"`
